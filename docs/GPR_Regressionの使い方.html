GPRによるB-Hカーブ回帰プログラム 取扱説明書

このドキュメントは、ガウス過程回帰（GPR）を用いてB-Hカーブを予測するPythonプログラムの操作方法、入出力ファイル、および各種設定について詳細に説明します。

1. ファイルパスの設定

本プログラムは、実行するスクリプトの場所を基準とした「相対パス」で動作します。これにより、親フォルダごと移動させても正しく動作します。

1.1. 入力ファイル

プログラムの実行に必要な入力ファイルは、以下のコード行でパスが設定されています。

ファイルの種類 説明 設定されているコード行
Akima補間データ B-Hループの先端部分（磁化曲線の頂点）を学習させるためのデータ。 
akima_excel_path = os.path.join(...)
学習データ GPRモデルがB-Hループの形状を学習するための、複数の振幅における実測データ群。 
input_base = os.path.join(...)
正解データ 予測結果の精度（RMSE）を計算するための、比較対象となる実測データ。 
truth_data_base = os.path.join(...)

1.2. 出力ファイル

プログラムの実行結果が出力されるファイルは、以下のコード行でパスが設定されています。

ファイルの種類 説明 設定されているコード行
ハイパーパラメータ 学習によって得られたGPRモデルのパラメータを保存します。学習時の全設定も`Info`シートに記録されます。 
hyper_param_dir = os.path.join(...)
最終結果 RMSEサマリー、各振幅でのグラフと比較データなど、実行結果のすべてを格納します。 
output_base = os.path.join(...)

2. プログラムの各種設定

スクリプト上部の「ユーザー設定箇所」で、プログラムの動作を詳細に制御できます。

2.1. 動作モード設定

PERFORM_TRAINING

モデルの学習を行うかどうかを決定する、最も重要なスイッチです。

  • True: 常にモデルの再学習を強制的に実行します。
  • False: 既存の学習結果（ハイパーパラメータファイル）を読み込みます。ただし、他の設定が変更されている場合は、安全のために自動で再学習が実行されます。

2.2. モデルと学習データ

どのようなモデルを、どのデータで学習させるかを設定します。

パラメータ 説明
mat_name 材料名（例: "50A470"）。フォルダ名やファイル名の一部として使用されます。
target_freq 対象周波数（例: 50）。
kernel_type GPRカーネルの種類（例: "Matern52"）。
train_amp 学習に使用するデータの振幅リスト（例: [0.4, 0.5, ..., 1.8]）。

2.3. 最適化の設定

モデルの学習（ハイパーパラメータ最適化）の挙動を制御します。

パラメータ 説明
OPTIMIZER 最適化アルゴリズムを選択します。"lbfgsb"（安定）または"scg"（高速な場合がある）から選びます。
MAX_ITERS 最適化計算1回あたりの最大繰り返し（イテレーション）回数です。
NUM_RESTARTS 最適化を異なる初期値からやり直す回数です。最も良い結果が採用されます。

2.4. 回帰対象の設定

学習済みモデルを使って、どのB-Hループを予測・生成するかを設定します。

パラメータ 説明
Bmreg_min 予測を開始する最小の磁束密度振幅（T）。
Bmreg_max 予測を終了する最大の磁束密度振幅（T）。
step 予測する振幅の間隔（T）。
上書き機能について: PERFORM_TRAINING = Falseで実行し、かつ上記2.2と2.3の設定が前回と完全に一致する場合、結果サマリーファイルは新規作成されず、既存の最新ファイルが削除され、新しい日時のファイル名で保存されます。
